# UI/画面設計書 — 政治シミュレーションゲーム

本ゲームのMVP段階における画面構成・画面遷移・各画面のレイアウトと操作を定義する。具体的なビジュアルデザイン (色, フォント, アイコン等) はアーキテクチャ設計書または実装フェーズで決定する。

## 1. 設計方針

- **テキスト中心UI**: MVPでは文字情報を主とし、グラフやチャートは将来拡張で対応する
- **情報密度のバランス**: 主要な状態 (ターン・指標・議席・資金) は常に視界に入る位置に固定配置する
- **操作の最小化**: 1画面でユーザーが取れる主要アクションは原則1〜3個に絞り、迷いを減らす
- **モバイル考慮**: 画面幅375px以上で快適に操作できるレイアウトとする
- **キーボード操作**: 主要操作はキーボードでも完結できる

## 2. 共通レイアウト

全画面で以下の構造を共通とする。

```
┌─ ヘッダー ─────────────────────────────────┐
│ [ターン情報] | [プレイヤー党と立場]         │
│ 議席: 現在/465  党資金: ○億円              │
├─ ステータスバー ───────────────────────────┤
│ 支持率45  経済52  財政38  外交安保60  環境55│
├─ メインエリア ─────────────────────────────┤
│                                             │
│   (画面ごとに異なる内容)                    │
│                                             │
├─ フッター ─────────────────────────────────┤
│ [政党情報] [ヘルプ] [メニュー]    [次へ →]│
└─────────────────────────────────────────────┘
```

### 2.1 ヘッダー

| 要素 | 内容 |
|---|---|
| ターン情報 | 「3年目前半 (2027年1-6月) / 通常国会」のような表記 |
| プレイヤー党 | 党名と立場 (野党第一党 / 連立与党 / 単独与党 / 少数与党) |
| 議席数 | 現在の議席 / 衆議院定数465 |
| 党資金 | 現在の党資金 |

### 2.2 ステータスバー

5指標 (支持率・経済・財政・外交安保・環境) を常時表示する。各指標には前ターンからの変化を示す矢印 (↑↓) を併記する。

### 2.3 フッター

| 要素 | 動作 |
|---|---|
| 政党情報 | 政党情報画面 (SC-10) へ遷移 |
| ヘルプ | ヘルプ画面 (SC-11) へ遷移 |
| メニュー | メニューモーダルを表示 (タイトルに戻る等) |
| 次へ | 現在のフェーズを完了して次のフェーズへ進む |

### 2.4 例外

タイトル画面 (SC-01) と党設定画面 (SC-02) では、ヘッダー・ステータスバーは非表示。エンディング画面 (SC-09) ではフッターの「次へ」ボタンを「タイトルへ戻る」に変更する。

## 4. 画面遷移

### 4.1 全体フロー

```
SC-01 (タイトル)
   ↓ 新規ゲーム
SC-02 (党設定)
   ↓ 設定完了
SC-03 (ダッシュボード) ←─────────────────┐
   ↓ イベントフェーズ                    │
SC-04 (イベント) → 対応完了 → SC-03      │
   ↓ 政策フェーズ                        │
SC-05 (政策選択) → 確定 → SC-03          │
   ↓ 予算フェーズ                        │
SC-06 (予算配分) → 確定 → SC-03          │
   ↓ (党運営フェーズはダッシュボードで)  │
   ↓ 指標更新                            │
   ↓ (選挙ターンの場合のみ)              │
SC-07 (選挙結果)                         │
   ↓ (過半数未達の場合)                  │
SC-08 (連立構成) → 確定 → SC-03 (次ターン) ──┘

最終ターン到達 or 特殊終了
   ↓
SC-09 (エンディング)
   ↓ タイトルへ戻る
SC-01
```

横断遷移として、ダッシュボードや各画面のフッターから SC-10 (政党情報) と SC-11 (ヘルプ) にいつでも遷移できる。

## 5. 各画面の詳細

### 5.1 SC-01: タイトル画面

**目的**: ゲームの入口。新規ゲーム開始, ヘルプ参照を行う。

**表示情報**:

- ゲームタイトル
- バージョン情報

**操作**:

- 「新規ゲーム」ボタン → SC-02 へ
- 「ヘルプ」ボタン → SC-11 へ

**レイアウト**:

```
┌─────────────────────────────────────┐
│                                     │
│      政治シミュレーションゲーム      │
│        (仮称)                       │
│                                     │
│       [ 新規ゲーム ]                │
│       [ ヘルプ ]                    │
│                                     │
│         version 0.1.0               │
└─────────────────────────────────────┘
```

### 5.2 SC-02: 党設定画面

**目的**: プレイヤーが党名と党のイデオロギー (3軸) を設定する。

**表示情報**:

- 党名入力欄
- 経済軸 / 社会軸 / 外交軸のセレクタ (-2〜+2)
- 各軸の説明文
- 設定済みイデオロギーから推測されるポジショニング (例: 「保守・自主路線寄り」)

**操作**:

- 党名入力
- 各軸の値選択
- 「決定」ボタン → SC-03 へ
- 「タイトルに戻る」ボタン → SC-01 へ

**レイアウト**:

```
┌─ 党設定 ──────────────────────────────┐
│ 党名: [____________________]         │
│                                       │
│ イデオロギー設定                       │
│   経済軸: ◯小さな政府 ─●─ 大きな政府  │
│   社会軸: ◯保守 ──●── リベラル      │
│   外交軸: ◯同盟重視 ──●── 自主路線   │
│                                       │
│ 推測されるポジショニング:              │
│   「中道リベラル・自主路線寄り」       │
│                                       │
│ [タイトルへ戻る]      [ 決定 ]       │
└───────────────────────────────────────┘
```

**バリデーション**:

- 党名が空 → エラー表示
- 党名がNPC党 (民自党, 公明会, 改革維新党, 立憲党, 人民党) と重複 → 警告表示
- 全軸が0 → 「特徴のない党」警告 (続行は可能)

### 5.3 SC-03: メインダッシュボード

**目的**: 現在のターンの状況を把握し、各フェーズに進む起点となる画面。

**表示情報**:

- ターン情報 (ヘッダー)
- 全指標 (ステータスバー)
- 当ターンに発生する予定の定例イベント (もしあれば)
- 進行中の連鎖イベント (もしあれば)
- 今ターンのフェーズ進行状況 (どこまで完了したか)
- 主要なトレンドの進行度 (簡易表示)
- 党結束度・連立健全度

**操作**:

- 「次のフェーズへ」ボタン → 該当フェーズの画面へ
- フッターからの横断遷移

**レイアウト**:

```
┌─ ヘッダー / ステータスバー ─────────────┐
├─────────────────────────────────────────┤
│ 今ターン: 3年目前半 (2027年1-6月)        │
│ 国会: 通常国会期間 (本予算審議)          │
│                                          │
│ フェーズ進行                              │
│   ✓ イベント   ▶ 政策   - 予算   - 党運営│
│                                          │
│ 党の内部状態                              │
│   党結束度: 62 / 連立健全度: ―          │
│                                          │
│ トレンドの進行                            │
│   少子高齢化: ▮▮▮▮▯▯▯▯▯▯ 35           │
│   気候変動:   ▮▮▮▯▯▯▯▯▯▯ 28           │
│   米中対立:   ▮▮▮▮▮▯▯▯▯▯ 45           │
│   AI技術革新: ▮▮▯▯▯▯▯▯▯▯ 18           │
│   地方衰退:   ▮▮▮▮▯▯▯▯▯▯ 38           │
│                                          │
│ 解散総選挙: (与党時のみ) [解散を判断する]│
├─ フッター ──────────────────────────────┤
│ [政党情報] [ヘルプ] [メニュー] [次フェーズへ] │
└─────────────────────────────────────────┘
```

### 5.4 SC-04: イベント画面

**目的**: 発生したイベントの内容を提示し、プレイヤーが対応を選択する。

**表示情報**:

- イベント名とカテゴリ
- イベントの説明文
- 即時効果 (もしあれば)
- 選択肢 (0〜4個) と各選択肢の効果

**操作**:

- 選択肢のいずれかを選んで「決定」ボタン
- 選択肢がない場合は「了解」ボタンで次へ

**レイアウト**:

```
┌─ ヘッダー / ステータスバー ─────────────┐
├─────────────────────────────────────────┤
│ 【国際情勢】尖閣諸島周辺で中国船が領海侵入│
│                                          │
│ 中国海警局の船が日本の領海に侵入した。   │
│ 対応を迫られている。                     │
│                                          │
│ ▶ 対応を選択                            │
│ ◯ A: 海保増派・対米連携要請              │
│      支持率+5 / 対米+ / 対中-             │
│ ◯ B: 外交ルートで強く抗議のみ             │
│      支持率-3 / 対中+1                    │
│ ◯ C: 静観・冷静対応                       │
│      支持率-8 / 外交安保-                 │
│                                          │
├─ フッター ──────────────────────────────┤
│ [政党情報] [ヘルプ] [メニュー]   [決定] │
└─────────────────────────────────────────┘
```

**特殊ケース**:

- 即時効果のみのイベント (選択肢なし) は「了解」ボタンのみ
- 連鎖イベントが発生する場合は次ターンの予告を画面下部に表示

### 5.5 SC-05: 政策選択画面

**目的**: プレイヤーが今ターン提出する政策を1〜3本選択する。

**表示情報**:

- カテゴリ別の政策リスト (48本)
- 各政策の名前, 重要度, 即効性, 主な効果
- 政策ごとの必要議席率と通過予想 (現状の議席で通るかの目安)
- 選択中の政策リスト (上限3本)

**操作**:

- カテゴリタブで絞り込み
- 政策をクリックで詳細表示
- 「選択に追加」「選択から外す」
- 「提出」ボタン → 通過判定を実施 → SC-03 へ

**レイアウト**:

```
┌─ ヘッダー / ステータスバー ─────────────┐
├─────────────────────────────────────────┤
│ 政策フェーズ - 今ターンの提出政策を選択  │
│                                          │
│ [税制財政|経済産業|社会保障|教育|環境|外交|行政]│
│                                          │
│ ── 税制財政 ──                           │
│ ◯ 消費税減税 (重要度2, 即時)             │
│   支持率++/経済+/財政-- |通過予想: 通過│
│ ◯ 消費税増税 (重要度2, 即時)             │
│   支持率--/財政++/経済- |通過予想: 否決│
│ ◯ 法人税減税 (重要度2, 即時)             │
│   経済+/財政-/...      |通過予想: 通過│
│ ... (他の政策)                          │
│                                          │
│ 選択中 (2/3): [消費税減税] [規制緩和]    │
│                                          │
├─ フッター ──────────────────────────────┤
│ [政党情報] [ヘルプ] [メニュー]   [提出] │
└─────────────────────────────────────────┘
```

**詳細モーダル**: 政策をクリックすると、より詳しい説明と効果値を表示するモーダルが開く。

### 5.6 SC-06: 予算配分画面

**目的**: 本予算 (前半ターン) または補正予算 (後半ターン) の配分を決定する。

**表示情報**:

- 予算タイプ (本予算 / 補正予算)
- 歳入額 (本予算の場合)
- 主要6分野の配分スライダーまたは数値入力
- 各分野への配分から予想される指標への影響
- 合計額と歳入との差分

**操作**:

- 各分野の配分を調整
- 合計額の確認
- 「提出」ボタン → 効果反映 → SC-03 へ
- 補正予算の場合は「組まずに進む」ボタンも表示

**レイアウト**:

```
┌─ ヘッダー / ステータスバー ─────────────┐
├─────────────────────────────────────────┤
│ 予算フェーズ - 本予算編成                │
│ 歳入: 110兆円  (許容赤字: +30兆円まで)   │
│                                          │
│ 配分                                      │
│   税制財政:       ─●─ 10兆円  → 財政+  │
│   経済産業:       ──●─ 15兆円  → 経済+│
│   社会保障:       ───●─ 35兆円 → 支持+ │
│   教育科技:       ──●── 10兆円 → 経済+│
│   環境エネルギー: ─●─── 8兆円  → 環境+│
│   外交防衛:       ──●── 12兆円 → 安保+│
│                                          │
│ 合計: 90兆円 (歳入: 110兆円)             │
│ 残り: 20兆円                              │
│                                          │
├─ フッター ──────────────────────────────┤
│ [政党情報] [ヘルプ] [メニュー]   [提出] │
└─────────────────────────────────────────┘
```

**補正予算モード**: 後半ターンの場合は「補正予算を組むか」のトグルを最上部に表示し、組む場合のみ配分UIを展開する。

### 5.7 SC-07: 選挙結果画面

**目的**: 選挙の結果を発表し、各党の議席分布を提示する。

**表示情報**:

- 選挙前後の議席比較
- プレイヤー党の獲得議席
- 全党の議席分布 (バーグラフ風)
- 過半数達成の有無
- 地方別の議席増減 (簡易表示。`ElectionResult.seatsPerParty` と前回値から導出)

(投票率の表示は将来拡張。MVPの `ElectionResult` には投票率フィールドを持たない)

**操作**:

- 「次へ」ボタン → 過半数未達なら SC-08 (連立構成) へ, それ以外は SC-03 へ

### 5.9 SC-09: エンディング画面

**目的**: ゲーム終了時の最終スコアと称号を表示する。

## 6. ダイアログ・モーダル

主要な確認・補足表示はダイアログ (モーダル) で行う。

| ID | 名称 | 起動タイミング |
|---|---|---|
| DG-01 | 解散総選挙の確認 | 与党時, プレイヤーが解散を選択時 |
| DG-02 | 政策の詳細 | 政策選択画面で政策をクリック時 |
| DG-03 | ターン終了の確認 | 全フェーズ完了時 |
| DG-04 | ゲーム終了の確認 | メニューからタイトルへ戻る時 |
| DG-05 | エラー通知 | 入力エラー・歳入超過等 |

各ダイアログは中央配置のオーバーレイで表示し、外側をクリックで閉じる (確認系は明示的なボタンのみ閉じられる)。

## 7. 操作系統の方針

### 7.1 マウス・タッチ操作

- 全画面の主要操作はマウスクリック / タッチで完結
- ボタンの最小タップ領域は44x44px (モバイルアクセシビリティ準拠)

### 7.2 キーボード操作

| キー | 動作 |
|---|---|
| Tab | フォーカス移動 |
| Enter | フォーカス中の要素を実行 |
| Esc | モーダルを閉じる, または前の画面に戻る |
| 数字キー (1-9) | 該当する選択肢 (イベント等) を選択 |
| 矢印キー | リスト内の移動 |

### 7.3 アクセシビリティ

- 全インタラクティブ要素は適切な aria-label を持つ
- 重要な情報を色だけで表現しない (アイコン・テキストを併用)
- フォーカスの可視化を明確にする

## 8. レスポンシブ対応

### 8.1 ブレイクポイント

| 区分 | 画面幅 | レイアウト |
|---|---|---|
| デスクトップ | 1024px以上 | サイドバー付きの2カラム可能 |
| タブレット | 768px〜1023px | 1カラム, ヘッダー固定 |
| モバイル | 375px〜767px | 1カラム, ヘッダー圧縮表示 |

### 8.2 モバイルでの調整

- ステータスバーの指標表示は横スクロール可能にする
- 政策選択画面のカテゴリタブは横スクロールで提示
- 予算配分のスライダーはタップ操作に最適化
- ボタンは画面下部に固定配置 (Sticky Footer) で常に押しやすい位置に

### 8.3 縦持ち・横持ち

モバイルでは縦持ちを基本とする。横持ちでも崩れないようにレイアウトを CSS Grid または Flexbox で構築する。
