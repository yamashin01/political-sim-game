# UI/画面設計書 — 政治シミュレーションゲーム

本ゲームのMVP段階における画面構成・画面遷移・各画面のレイアウトと操作を定義する。具体的なビジュアルデザイン (色, フォント, アイコン等) はアーキテクチャ設計書または実装フェーズで決定する。

## 1. 設計方針

- **テキスト中心UI**: MVPでは文字情報を主とし、グラフやチャートは将来拡張で対応する
- **情報密度のバランス**: 主要な状態 (ターン・指標・議席・資金) は常に視界に入る位置に固定配置する
- **操作の最小化**: 1画面でユーザーが取れる主要アクションは原則1〜3個に絞り、迷いを減らす
- **モバイル考慮**: 画面幅375px以上で快適に操作できるレイアウトとする
- **キーボード操作**: 主要操作はキーボードでも完結できる

## 2. 共通レイアウト

全画面で以下の構造を共通とする。

```
┌─ ヘッダー ─────────────────────────────────┐
│ [ターン情報] | [プレイヤー党と立場]         │
│ 議席: 現在/465  党資金: ○億円              │
├─ ステータスバー ───────────────────────────┤
│ 支持率45  経済52  財政38  外交安保60  環境55│
├─ メインエリア ─────────────────────────────┤
│                                             │
│   (画面ごとに異なる内容)                    │
│                                             │
├─ フッター ─────────────────────────────────┤
│ [政党情報] [ヘルプ] [メニュー]    [次へ →]│
└─────────────────────────────────────────────┘
```

### 2.1 ヘッダー

| 要素 | 内容 |
|---|---|
| ターン情報 | 「3年目前半 (2027年1-6月) / 通常国会」のような表記 |
| プレイヤー党 | 党名と立場 (野党第一党 / 連立与党 / 単独与党 / 少数与党) |
| 議席数 | 現在の議席 / 衆議院定数465 |
| 党資金 | 現在の党資金 |

### 2.2 ステータスバー

5指標 (支持率・経済・財政・外交安保・環境) を常時表示する。各指標の表示は以下の3要素で構成される。

| 要素 | 内容 |
|---|---|
| 指標ラベル + 現在値 | 数値は `font-mono tabular` で大きく表示 |
| 前ターン比デルタ | `▲+3` / `▼-2` / `─` の3パターン。改善方向は `--ink`、悪化方向は `--vermilion` で配色 |
| 推移スパークライン | 過去最大6ターンの推移を inline SVG で表示。線色は `--ink/40`、最新点に `--vermilion` ドット |

スパークラインのデータソースは `state.history.turns[].indicatorsAtEnd` (各ターン末のスナップショット) を直近6件取り出して構築する。初回ターンなど履歴が0件の場合はスパークラインを描画せず、デルタも `─` 表示とする。

デルタ値は `gameStore.lastCompletedTurnDelta` を参照する (直近で完了したターンの指標変化を保持する派生値)。初回ターンや advance 直前は `null` のため、UI 側で `─` フォールバックする。

### 2.3 フッター

| 要素 | 動作 |
|---|---|
| 政党情報 | 政党情報画面 (SC-10) へ遷移 |
| ヘルプ | ヘルプ画面 (SC-11) へ遷移 |
| メニュー | メニューモーダルを表示 (タイトルに戻る等) |
| 次へ | 現在のフェーズを完了して次のフェーズへ進む |

### 2.4 説明ボックス

各画面のメインエリア最上部に「説明ボックス」を常時表示し、その画面で何ができるか・どんな判断を行うかを 1〜3 文で示す。ユーザーが各画面に到達したときに迷わないようにすることが目的。

```
┌───────────────────────────────────────┐
│【 解説 】[画面タイトル / フェーズ名]  │ ← 黒帯のキッカー (smallcaps)
├───────────────────────────────────────┤
│ ・                                     │
│ │  [この画面で何ができるか・何を判断するか]
│ │  [必要に応じて、操作のヒントや結果の流れ]
│ ・                                     │
└───────────────────────────────────────┘
```

| 項目 | 仕様 |
|---|---|
| 配置 | メインエリア最上部 (ヘッダー・ステータスバーの直下) |
| 配色 | 紙地 (`--paper`) + 上下罫線。上部に黒帯 (`--ink`) のキッカー (カテゴリ + タイトル) |
| キッカー | 「【 解説 】」「【 編集主幹より 】」など、画面ごとに切替可 (props 経由) |
| アクセント | 左側に縦の細罫 + 上下に朱 (`--vermilion`) の点を配置 |
| 文体 | 丁寧・解説調 (敬体)。設計書の文章トーンに合わせる |
| 文量 | 原則 1〜3 文。情報過多にしない |
| 対象画面 | SC-02 〜 SC-08 (タイトル, エンディング, 政党情報, ヘルプは対象外) |

説明ボックスはユーザーが閉じることはできない (MVP では常時表示)。将来的に「もう表示しない」トグルを設ける可能性はあるが、本MVPでは対象外。

### 2.5 ツールチップ補足

説明ボックスに収まらない用語レベルの補足は、該当要素の脇に脚注マーカー `※` を配置し、クリック / タップでポップオーバーを開く。新聞紙面の脚注 (※印付き欄外注釈) を踏襲したアフォーダンスである。

| 項目 | 仕様 |
|---|---|
| マーカー | `※` (朱色)。フォントは `--font-display` を使用 |
| トリガー | マーカーへのクリック / タップでトグル開閉 |
| 閉じる | (1) マーカーを再クリック、(2) 外側クリック、(3) Esc キー |
| 表示位置 | 対象要素の直下中央。`max-width: calc(100vw - 2rem)` で画面端からの最小マージンを確保 |
| アクセシビリティ | `aria-describedby` でマーカー (button) と ポップオーバーを関連付け。開閉状態は `aria-expanded` で表現 |
| 内容 | 1〜2 文の用語説明。詳細はヘルプ画面 (SC-11) に誘導 |
| 対象用語 | 各画面の詳細 (§5) の「ツールチップ対象」欄に列挙する |

ホバーで自動的に開く設計は採用していない (タッチデバイスで再現できないため統一性を欠くため)。完全な衝突回避 (画面端でのフリップ等) は将来 Radix Popover 等への置換で対応する。

### 2.6 例外

タイトル画面 (SC-01) と党設定画面 (SC-02) では、ヘッダー・ステータスバーは非表示。エンディング画面 (SC-09) ではフッターの「次へ」ボタンを「タイトルへ戻る」に変更する。

## 4. 画面遷移

### 4.1 全体フロー

```
SC-01 (タイトル)
   ↓ 新規ゲーム
SC-02 (党設定)
   ↓ 設定完了
SC-03 (ダッシュボード) ←──────────────────────────┐
   ↓ イベントフェーズ                             │
SC-04 (イベント) → 対応完了 → SC-03               │
   ↓ 政策フェーズ                                 │
SC-05 (政策選択) → 確定 → SC-03                   │
   ↓ 予算フェーズ                                 │
SC-06 (予算配分) → 確定 → SC-03                   │
   ↓ (党運営フェーズはダッシュボードで)           │
   ↓ 指標更新                                     │
   ↓                                              │
SC-12 (ターン終了サマリ)                          │
   ↓ 「次号へ」                                   │
   ↓ (選挙ターンの場合のみ)                       │
SC-07 (選挙結果)                                  │
   ↓ (過半数未達の場合)                           │
SC-08 (連立構成) → 確定 → SC-03 (次ターン) ───────┘

最終ターン到達 or 特殊終了
   ↓
SC-09 (エンディング)
   ↓ タイトルへ戻る
SC-01
```

横断遷移として、ダッシュボードや各画面のフッターから SC-10 (政党情報) と SC-11 (ヘルプ) にいつでも遷移できる。

SC-12 (ターン終了サマリ) は SC-03 の党運営フェーズ完了時 (指標更新の直後) に必ず挿入される。「次号へ」ボタンが、選挙発動判定・連鎖イベント抽選・ターン番号繰上げの起点となる (MVP までは SC-03 がこの責務を担っていたが、視覚的な総括フィードバックを与えるため SC-12 に切り出した)。

## 5. 各画面の詳細

### 5.1 SC-01: タイトル画面

**目的**: ゲームの入口。新規ゲーム開始, ヘルプ参照を行う。

**表示情報**:

- ゲームタイトル
- バージョン情報

**操作**:

- 「新規ゲーム」ボタン → SC-02 へ
- 「ヘルプ」ボタン → SC-11 へ

**レイアウト**:

```
┌─────────────────────────────────────┐
│                                     │
│      政治シミュレーションゲーム      │
│        (仮称)                       │
│                                     │
│       [ 新規ゲーム ]                │
│       [ ヘルプ ]                    │
│                                     │
│         version 0.1.0               │
└─────────────────────────────────────┘
```

### 5.2 SC-02: 党設定画面

**目的**: プレイヤーが党名と党のイデオロギー (3軸) を設定する。

**説明ボックス**:

> あなたが率いる党の党名とイデオロギーを設定します。経済・社会・外交の3つの軸で立ち位置を決めると、ゲーム中の連立交渉や政策通過の判定に影響します。設定はゲーム開始後は変更できません。

**ツールチップ対象**:

| 用語 | 補足内容 (要旨) |
|---|---|
| 経済軸 | -2 (小さな政府) 〜 +2 (大きな政府)。財政・経済政策の親和性に影響 |
| 社会軸 | -2 (保守) 〜 +2 (リベラル)。社会保障や教育政策の親和性に影響 |
| 外交軸 | -2 (同盟重視) 〜 +2 (自主路線)。外交安保政策の親和性に影響 |
| ポジショニング | 3軸の組み合わせから推測される党の立ち位置の呼称 |

**表示情報**:

- 党名入力欄
- 経済軸 / 社会軸 / 外交軸のセレクタ (-2〜+2)
- 各軸の説明文
- 設定済みイデオロギーから推測されるポジショニング (例: 「保守・自主路線寄り」)

**操作**:

- 党名入力
- 各軸の値選択
- 「決定」ボタン → SC-03 へ
- 「タイトルに戻る」ボタン → SC-01 へ

**レイアウト**:

```
┌─ 党設定 ──────────────────────────────┐
│ 党名: [____________________]         │
│                                       │
│ イデオロギー設定                       │
│   経済軸: ◯小さな政府 ─●─ 大きな政府  │
│   社会軸: ◯保守 ──●── リベラル      │
│   外交軸: ◯同盟重視 ──●── 自主路線   │
│                                       │
│ 推測されるポジショニング:              │
│   「中道リベラル・自主路線寄り」       │
│                                       │
│ [タイトルへ戻る]      [ 決定 ]       │
└───────────────────────────────────────┘
```

**バリデーション**:

- 党名が空 → エラー表示
- 党名がNPC党 (民自党, 公明会, 改革維新党, 立憲党, 人民党) と重複 → 警告表示
- 全軸が0 → 「特徴のない党」警告 (続行は可能)

### 5.3 SC-03: メインダッシュボード

**目的**: 現在のターンの状況を把握し、各フェーズに進む起点となる画面。

**説明ボックス**:

> 今ターンの状況を確認し、各フェーズを順に進めます。1ターンは「イベント → 政策 → 予算 → 党運営」の4フェーズで構成されています。指標やトレンドの推移を見ながら、次にどのフェーズへ進むかを判断してください。

**ツールチップ対象**:

| 用語 | 補足内容 (要旨) |
|---|---|
| 党結束度 | 党内の意見の一致度。低下すると党分裂のリスクが高まる |
| 連立健全度 | 連立与党時のみ。低下すると連立崩壊のリスクが高まる |
| トレンド | 中長期で進行する社会課題。進行度100で大きな影響イベントが発生 |
| 解散総選挙 | 与党時のみ可能。任意のタイミングで衆議院選挙を実施できる |

**表示情報**:

- ターン情報 (ヘッダー)
- 全指標 (ステータスバー — 前ターン比デルタとスパークラインを含む)
- 当ターンに発生する予定の定例イベント (もしあれば)
- 進行中の連鎖イベント (もしあれば)
- 今ターンのフェーズ進行状況 (どこまで完了したか)
- 主要なトレンドの進行度 (進行度バー + 前ターン比デルタ)
- 支持率の推移チャート (右カラム、過去最大6ターンの折れ線グラフ)
- 党結束度・連立健全度

**支持率推移チャート**:

| 項目 | 仕様 |
|---|---|
| データソース | `state.history.turns[].indicatorsAtEnd.approval` の直近6件 + 現在値 |
| 描画 | recharts の `LineChart` を `ResponsiveContainer` でラップ |
| 線色 | `--vermilion` (主役の指標として強調) |
| 軸 | x軸はターン番号、y軸は 0〜100 固定 |
| 履歴ゼロ時 | 「— 記録なし」と表示してチャートは省略 |

**トレンドリストのデルタ**:

各トレンド行に「前ターン比 +X.X」を追加表示する。前ターンの進行度は `state.history.turns[]` の最新エントリから取得する (記録がない初回ターンは省略)。

**操作**:

- 「次のフェーズへ」ボタン → 該当フェーズの画面へ
- フッターからの横断遷移

**レイアウト**:

```
┌─ ヘッダー / ステータスバー ─────────────┐
├─────────────────────────────────────────┤
│ 今ターン: 3年目前半 (2027年1-6月)        │
│ 国会: 通常国会期間 (本予算審議)          │
│                                          │
│ フェーズ進行                              │
│   ✓ イベント   ▶ 政策   - 予算   - 党運営│
│                                          │
│ 党の内部状態                              │
│   党結束度: 62 / 連立健全度: ―          │
│                                          │
│ トレンドの進行                            │
│   少子高齢化: ▮▮▮▮▯▯▯▯▯▯ 35           │
│   気候変動:   ▮▮▮▯▯▯▯▯▯▯ 28           │
│   米中対立:   ▮▮▮▮▮▯▯▯▯▯ 45           │
│   AI技術革新: ▮▮▯▯▯▯▯▯▯▯ 18           │
│   地方衰退:   ▮▮▮▮▯▯▯▯▯▯ 38           │
│                                          │
│ 解散総選挙: (与党時のみ) [解散を判断する]│
├─ フッター ──────────────────────────────┤
│ [政党情報] [ヘルプ] [メニュー] [次フェーズへ] │
└─────────────────────────────────────────┘
```

### 5.4 SC-04: イベント画面

**目的**: 発生したイベントの内容を提示し、プレイヤーが対応を選択する。

**説明ボックス**:

> 今ターン発生したイベントへの対応を選びます。各選択肢には即時効果があり、ものによっては次ターン以降の連鎖イベントを引き起こします。選んだ対応は支持率や外交安保などの指標に影響します。

**ツールチップ対象**:

| 用語 | 補足内容 (要旨) |
|---|---|
| 即時効果 | この選択肢を選んだ瞬間に指標へ反映される変化 |
| 連鎖イベント | 次ターン以降に派生して発生するフォローアップ事象 |
| 選択肢の効果記号 | `+` は上昇、`-` は低下、`++` `--` は大幅変化を表す |

**表示情報**:

- イベント名とカテゴリ
- イベントの説明文
- 即時効果 (もしあれば)
- 選択肢 (0〜4個) と各選択肢の効果

**操作**:

- 選択肢のいずれかを選んで「決定」ボタン
- 選択肢がない場合は「了解」ボタンで次へ

**レイアウト**:

```
┌─ ヘッダー / ステータスバー ─────────────┐
├─────────────────────────────────────────┤
│ 【国際情勢】尖閣諸島周辺で中国船が領海侵入│
│                                          │
│ 中国海警局の船が日本の領海に侵入した。   │
│ 対応を迫られている。                     │
│                                          │
│ ▶ 対応を選択                            │
│ ◯ A: 海保増派・対米連携要請              │
│      支持率+5 / 対米+ / 対中-             │
│ ◯ B: 外交ルートで強く抗議のみ             │
│      支持率-3 / 対中+1                    │
│ ◯ C: 静観・冷静対応                       │
│      支持率-8 / 外交安保-                 │
│                                          │
├─ フッター ──────────────────────────────┤
│ [政党情報] [ヘルプ] [メニュー]   [決定] │
└─────────────────────────────────────────┘
```

**特殊ケース**:

- 即時効果のみのイベント (選択肢なし) は「了解」ボタンのみ
- 連鎖イベントが発生する場合は次ターンの予告を画面下部に表示

### 5.5 SC-05: 政策選択画面

**目的**: プレイヤーが今ターン提出する政策を1〜3本選択する。

**説明ボックス**:

> 今ターン提出する政策を最大3本まで選びます。党の議席率と他党の協力度合いで通過判定が行われ、通過した政策のみが指標に効果を与えます。重要な政策ほど通過に必要な議席率が高くなります。

**ツールチップ対象**:

| 用語 | 補足内容 (要旨) |
|---|---|
| 重要度 | 1〜3。値が大きいほど効果も大きいが通過に必要な議席率も高い |
| 即効性 | 「即時」は今ターンから、「遅延」は数ターン後から効果が現れる |

### 5.12 SC-12: ターン終了サマリ画面

**目的**: 当ターンの政策結果・対応イベント・指標変化・トレンド進行を「政局報 号外」スタイルの一画面で総括し、プレイヤーに「自分の判断が国家にどう作用したか」を視覚的に伝える。SC-03 の党運営フェーズ完了直後に必ず挿入される中間画面。

**配置の必然性**: 党運営フェーズで `applyTurnIndicators` が走った直後、選挙発動判定・次ターン繰上げの前にプレイヤーへ「総括の間」を設ける。MVP 段階ではこの間が存在せず、プレイヤーが選択の影響を体感できないという課題があった。

**説明ボックス**:

> 当ターンに起きたことを総括する「号外」です。提出した政策の通過状況、対応したイベント、指標の変化、トレンドの進行を1ページで確認できます。「次号へ」を押すと選挙判定や次ターンの準備に進みます。

**ツールチップ対象**:

| 用語 | 補足内容 (要旨) |
|---|---|
| 指標の変化 | ターン開始時 → 終了時の差分。政策・イベント・トレンドの累積効果 |
| トレンド進行 | 対抗政策の有無で進行が抑制される。100到達で大型イベント |

**表示情報**:

| セクション | 内容 |
|---|---|
| 一面 (見出し) | 「号外」スタンプ + 「第N号 総括」大見出し + 在任ターン表記 |

| 経済面 (トレンド) | 各トレンドの進行度を Before → After で表示。対抗政策で抑制された場合は注記 |
| 次期予告 | 次ターンの予定 (定例イベント・選挙発動見込み確率など) を 1〜2 行で示唆 |

| DG-02 | 政策の詳細 | 政策選択画面で政策をクリック時 |
| DG-03 | ターン終了の確認 | 全フェーズ完了時 |
| DG-04 | ゲーム終了の確認 | メニューからタイトルへ戻る時 |
| DG-05 | エラー通知 | 入力エラー・歳入超過等 |

各ダイアログは中央配置のオーバーレイで表示し、外側をクリックで閉じる (確認系は明示的なボタンのみ閉じられる)。

## 7. 操作系統の方針

### 7.1 マウス・タッチ操作

- 全画面の主要操作はマウスクリック / タッチで完結
- ボタンの最小タップ領域は44x44px (モバイルアクセシビリティ準拠)

### 7.2 キーボード操作

| キー | 動作 |
|---|---|
| Tab | フォーカス移動 |
| Enter | フォーカス中の要素を実行 |
| Esc | モーダルを閉じる, または前の画面に戻る |
| 数字キー (1-9) | 該当する選択肢 (イベント等) を選択 |
| 矢印キー | リスト内の移動 |

### 7.3 アクセシビリティ

- 全インタラクティブ要素は適切な aria-label を持つ
- 重要な情報を色だけで表現しない (アイコン・テキストを併用)
- フォーカスの可視化を明確にする

## 8. レスポンシブ対応

### 8.1 ブレイクポイント

| 区分 | 画面幅 | レイアウト |
|---|---|---|
| デスクトップ | 1024px以上 | サイドバー付きの2カラム可能 |
| タブレット | 768px〜1023px | 1カラム, ヘッダー固定 |
| モバイル | 375px〜767px | 1カラム, ヘッダー圧縮表示 |

### 8.2 モバイルでの調整

- ステータスバーの指標表示は横スクロール可能にする
- 政策選択画面のカテゴリタブは横スクロールで提示
- 予算配分のスライダーはタップ操作に最適化
- ボタンは画面下部に固定配置 (Sticky Footer) で常に押しやすい位置に

### 8.3 縦持ち・横持ち

モバイルでは縦持ちを基本とする。横持ちでも崩れないようにレイアウトを CSS Grid または Flexbox で構築する。
