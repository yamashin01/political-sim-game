## 1. プロジェクト概要

### 1.1 背景

現代の日本社会において、政治への関心の低下が指摘されて久しい。多くの人にとって政治は「難しい」「自分には関係ない」「特定の党を支持するかどうかの選択」と捉えられがちであり、政治の構造そのもの — 政策のトレードオフ、選挙戦略、国際情勢の影響 — を学ぶ機会は限られている。

既存の政治学習コンテンツは教科書的で堅苦しいものが多く、また中立性を保つことが難しいため、特定の立場に偏った内容となりがちである。

### 1.2 プロジェクトの目的

本ゲームは、プレイヤーが**政党党首**として日本の政治運営を体験することで、政治の難しさと面白さを学べるWebシミュレーションゲームを提供する。

具体的には以下の体験をプレイヤーに提供する。

- 政策決定のトレードオフ (例: 減税で支持率は上がるが財政は悪化する) を自身の判断として体験する
- 外国勢力, 経済情勢, 自然災害など、自国だけでは制御できない要素との向き合い方を学ぶ
- 党のイデオロギー (理念) と現実主義 (人気取り) の緊張関係を体感する
- 短期の人気取りと長期の国益のジレンマに直面する

### 1.3 提供価値

| 価値 | 内容 |
|---|---|
| エンターテインメント | 戦略性とリプレイ性のあるゲームプレイ |
| 教育 | プレイ後に政治ニュースの語彙・論点が理解できるようになる |
| 自己発見 | 自分の政治観 (経済重視か, 環境重視か等) に気づくきっかけを与える |

### 1.4 設計の基本方針

- **シンプル : リアル = 7 : 3**: 完全なリアリティより、ゲームとしての分かりやすさを優先する。ただしトレードオフの構造はリアルに保つ
- **公平性**: 特定の政治的立場を持ち上げたり批判したりせず、複数の視点を等価に提示する
- **拡張性**: MVPは固定設定で実装するが、桃太郎電鉄のように設定を変えて何度も遊べる構造を内部に備える

## 2. スコープ

### 2.1 MVPで対応する範囲

| 項目 | 設定 |
|---|---|
| プレイ期間 | 10年固定 (20ターン) |
| 1ターンの単位 | 6ヶ月 |
| 選挙対象 | 衆議院議員総選挙のみ |
| プレイヤーのスタート位置 | 野党第一党 固定 |
| 勝利条件 | 総合スコア型 固定 |
| プレイヤー数 | 1人 (vs CPU政党) |
| シナリオ | 標準シナリオ (現代日本ベース) 1種類 |
| 難易度 | 単一 (ノーマル) |
| UI形式 | テキスト中心UI |
| 言語 | 日本語のみ |
| 対応プラットフォーム | Webブラウザ (PC, モバイル) |

### 2.2 MVP対象外 (将来拡張で対応)

以下の機能は将来のバージョンアップで対応する (詳細は第9章を参照)。

**ゲーム設定の拡張**
- プレイ期間のカスタマイズ
- 参議院選挙・地方選挙
- スタート位置の選択
- 多様な勝利条件
- マルチプレイヤー
- 複数シナリオ
- 難易度選択

**機能の拡張**
- 教育サポート機能 (政策・イベント注釈表示, 振り返り画面, 用語集)
- セーブ・ロード機能
- グラフ・チャートによる可視化の充実
- 多言語対応

**ゲームメカニクスの精緻化**
- 公約達成度システム (施政方針演説イベント含む)
- 候補者の質システム
- 選挙資金投入による議席への影響
- 累積効果システム (同種イベント連発による影響増幅)
- 派閥対立や選挙結果による党結束度の細やかな変動
- 連立交渉の譲歩補正

## 4. 機能要件

### 4.1 機能一覧

| 機能ID | 機能名 | 概要 |
|---|---|---|
| F-01 | ゲーム開始機能 | 党名・イデオロギーの設定。ゲーム初期化 |
| F-02 | ターン進行機能 | 各ターンのフェーズ管理 |
| F-03 | 政策提出機能 | 政策の選択と通過判定 |
| F-04 | イベント対応機能 | 発生イベントへの選択肢提示と対応 |
| F-05 | 予算配分機能 | 主要分野への予算割り振り (本予算・補正予算) |
| F-06 | 選挙機能 | 選挙の実施・議席数の確定・与党/野党判定 |
| F-07 | 連立交渉機能 | 過半数未満時の連立交渉 |
| F-08 | スコア算出・エンディング機能 | ゲーム終了時の最終評価と称号付与 |

### 4.2 各機能の概要

機能の詳細仕様 (入力・処理・出力・例外処理) は機能設計書で定義する。本書では機能の目的と概要のみ示す。

#### F-01: ゲーム開始機能

プレイヤーが党名と党のイデオロギー (経済軸・社会軸・外交軸の3軸) を設定し、新規ゲームを開始する。ゲームの初期状態 (議席数, 指標, 資金等) が生成される。

#### F-02: ターン進行機能

1ターン (6ヶ月) を以下のフェーズで進行させる。

```
イベントフェーズ → 政策フェーズ → 予算配分フェーズ
                → 党運営フェーズ → 指標更新 → (該当時) 選挙フェーズ
```

各フェーズでプレイヤーが必要な決定を行う。ターンは「前半 (1-6月, 通常国会期間)」と「後半 (7-12月, 臨時国会期間)」で性格が異なり、発生する定例イベントや予算の種類が変化する。

#### F-03: 政策提出機能

プレイヤーは1ターンに重要法案を1〜3本提出できる。各政策について、議席数と党イデオロギーの合致度から必要議席率を算出し、通過判定を行う。通過した政策の効果が指標に反映される。

#### F-04: イベント対応機能

各ターンのイベントフェーズで、ランダム・条件付きランダム・政策連動・時系列・連鎖のいずれかでイベントが発生する。プレイヤーは提示された選択肢から対応を選び、その効果を反映する。一部のイベントは選択肢なし (即時効果のみ) で発生する。

#### F-05: 予算配分機能

予算は以下の2種類を扱う。

- **本予算**: 通常国会期間 (前半ターン) に翌年度予算を編成・成立させる。主要6分野 (税制財政, 経済産業, 社会保障, 教育科技, 環境エネルギー, 外交防衛) への配分を決定する。歳入が上限となる。
- **補正予算**: 臨時国会期間 (後半ターン) に必要に応じて編成する。災害対応や経済ショック対策などの追加支出を判断する。

「平時の本予算 vs 非常時の補正予算」の使い分けがプレイヤーの判断ポイントとなる。

#### F-06: 選挙機能

野党時は1〜4年に1回ランダムで、与党時はプレイヤーが任期4年以内に解散時期を指定する形で、衆議院議員総選挙を実施する。各党の議席数を、地方支持率と経済情勢補正 (現職効果) から算出する。

与党時の解散総選挙のタイミング判断は戦略的に重要であり、支持率, 国際情勢の安定度, 経済指標の状況, 不信任案提出の動きなどを総合して判断する必要がある。

#### F-07: 連立交渉機能

選挙結果が過半数未満の場合、他党との連立を構成する。他党のイデオロギー座標との距離で連立成立率と連立健全度が変動する。MVPでは連立交渉は自動 (イデオロギー距離が近い党から順に過半数まで)。

#### F-08: スコア算出・エンディング機能

ゲーム終了時 (20ターン経過時, または特殊終了時) に総合スコアを算出する。5指標の最終値、党理念達成度、在任期間からスコアを計算し、称号 (歴史的名宰相, 経済再生の立役者など) を付与する。

## 5. ゲーム要素 (ドメイン要件)

本章ではゲームを構成するドメイン要素を一覧する。各要素の詳細スキーマはデータ設計書、数値はゲームバランス設計書で定義する。

### 5.1 ゲームの基本構造

| 項目 | 設定 |
|---|---|
| 期間 | 10年固定 |
| 1ターン | 6ヶ月 |
| 総ターン数 | 20ターン |
| 衆議院定数 | 465議席 |
| 過半数ライン | 233議席 |
| プレイヤー初期議席 | 約100〜120議席 (野党第一党想定) |

1ターン (6ヶ月) は現実の日本の国会サイクルに対応し、以下2種類の性格を持つ。

| ターン位置 | 対応月 | 性格 | 主な活動 |
|---|---|---|---|
| 前半 | 1月-6月 | 通常国会期間 (150日) | 翌年度本予算の編成・審議, 重要法案の本格審議, 解散判断 (与党時, 6月) |
| 後半 | 7月-12月 | 国会閉会・臨時国会期間 | 補正予算 (必要時), 概算要求, 国際情勢対応, 党運営 |

この区別により、ターンごとに異なる戦略的選択が求められる。前半は予算と内政、後半は外交と党運営が中心となる。

### 5.2 プレイヤーが管理するリソース

| リソース | 役割 |
|---|---|
| 議席数 | 政策通過可否を決める基盤 |
| 政策選択 | 各ターンで実行する政策の決定権 |
| 党資金 | 選挙活動・広告・候補者擁立・政策推進に使用 |

### 5.3 党のイデオロギー (3軸)

ゲーム開始時に設定する。設定後は原則固定 (変更時は大きなペナルティ)。

| 軸 | 範囲 |
|---|---|
| 経済軸 | -2 (小さな政府) 〜 +2 (大きな政府) |
| 社会軸 | -2 (保守) 〜 +2 (リベラル) |
| 外交軸 | -2 (同盟重視) 〜 +2 (自主路線) |

### 5.4 国家の指標 (5指標)

ゲーム全体を通じて自動更新される。プレイヤーは政策とイベント対応を通じて間接的にこれらを動かす。

| 指標 | 説明 |
|---|---|
| 支持率 | 民意。選挙結果に直結 |
| 経済 | GDP成長率と雇用の複合 |
| 財政 | 国債残高/GDP比 (低いほど健全) |
| 外交安保 | 国際関係と防衛の統合 (内部で対米/対中/対近隣3軸を保持) |
| 環境 | CO2・自然保護・再エネ比率の複合 |

### 5.5 政党構成 (6党制)

プレイヤーの党を含む6党制を採用する。実在の日本の政党をモデルにしつつ名称はぼかす。

| 政党名 | モデル | 立ち位置 |
|---|---|---|
| 民自党 | 自民党 | 保守・大企業寄り |
| 公明会 | 公明党 | 中道・福祉重視 |
| 改革維新党 | 維新の会 | 改革保守 |
| 立憲党 | 立憲民主党 | 中道左派 |
| 人民党 | 共産党+れいわ | 左派 |
| (プレイヤーの党) | — | 開始時に設定 |

### 5.6 地方ブロック (6ブロック)

地方ごとの温度差を表現するため6ブロックに分割する。各ブロックは独自の支持率と政策反応特性を持つ。

1. 北海道・東北
2. 関東
3. 中部
4. 関西
5. 中国・四国
6. 九州・沖縄

### 5.7 政策カタログ (全48本)

7カテゴリに分類された48本の政策を実装する。

| カテゴリ | 本数 |
|---|---|
| 税制財政 | 8 |
| 経済産業 | 8 |
| 社会保障 | 8 |
| 教育科技 | 6 |
| 環境エネルギー | 5 |
| 外交防衛 | 7 |
| 行政 | 6 |
| **合計** | **48** |

各政策の詳細スペック (効果値, 必要議席率, 地方補正, 副次効果等) はゲームバランス設計書で定義する。

### 5.8 イベントカテゴリ (全50本)

9カテゴリに分類された50本のイベントを実装する。

| カテゴリ | 本数 |
|---|---|
| 国際情勢 (政治・軍事) | 8 |
| 国際経済・エネルギー | 6 |
| 全国規模災害・社会 | 5 |
| 地方限定災害 | 5 |
| 政治スキャンダル | 6 |
| 経済ショック (国内発) | 4 |
| ポジティブイベント | 8 |
| 党内・政治構造突発 | 3 |
| 定例イベント (年中行事) | 5 |
| **合計** | **50** |

定例イベントは特定ターンに必ず発生するスケジュール固定イベント (トリガー種別: 時系列スクリプト) であり、以下を含む。

| 定例イベント | 発生時期 | 内容 |
|---|---|---|
| 予算委員会 | 前半ターン中 | 野党時は与党批判イベント, 与党時は答弁試練イベント |
| 党首討論 | 国会会期中 (前半・後半いずれか) | 党首力を試す中間イベント |
| G7サミット | 前半ターン終盤 (5-6月) | 外交アピールの機会。対米/対欧関係に影響 |
| 国連総会 | 後半ターン序盤 (9月) | 外交スピーチの機会。国際関係指標に影響 |
| 概算要求 | 後半ターン (8-9月) | 各省庁の予算要求が出揃う。予算編成方針の選択 |

### 5.9 トレンド (5本)

長期的に進行する変化。プレイヤーは止められず、政策で進行緩和や影響軽減を図る。

1. 少子高齢化
2. 気候変動
3. 米中対立
4. AI/技術革新
5. 地方衰退

### 5.10 状態 (2本)

継続的に変動するパラメータ。

| 状態 | 説明 |
|---|---|
| 党結束度 | 党内派閥の結束度 (0〜100) |
| 連立健全度 | 連立相手との関係 (与党時のみ, 0〜100) |

### 5.11 勝利条件と評価

10年経過時に総合スコアを算出し、エンディング種別を付与する。

スコア構成要素:

- 5指標の最終値
- 党理念達成度 (自党イデオロギーに沿った政策をどれだけ実行したか)
- 在任期間 (与党だった期間)

エンディング例: 歴史的名宰相 / 経済再生の立役者 / グリーンリーダー / 信念の人 / 老獪な政治家 / 失脚エンド

## 6. 非機能要件

### 6.1 性能要件

| 項目 | 目標 |
|---|---|
| 1ターン処理時間 | 2秒以内 |
| 画面遷移 | 1秒以内 |
| 初期ロード時間 | 5秒以内 |

### 6.2 ユーザビリティ要件

- 初心者がチュートリアルなしでもプレイ開始可能であること
- 主要な決定操作が3クリック以内で完了すること
- モバイル (画面幅375px〜) でも快適にプレイ可能なレイアウトであること
- 1セッションで完結できるプレイ時間 (1〜2時間程度) であること

### 6.3 拡張性要件

- 政策・イベント・トレンド・政党のデータを後から追加可能な構造とすること
- プレイ期間・選挙対象・勝利条件を Configuration として外部化し、設定変更だけで切り替え可能とすること
- 多言語化に備えて、UI文字列をリソース化すること
- 教育サポート機能・セーブ機能などの将来追加に備えて、データ構造に余白を残すこと

### 6.4 アクセシビリティ要件

- キーボード操作のみでプレイ可能であること
- 色覚特性に配慮した配色 (重要情報を色だけに依存しない)
- フォントサイズの調整機能
- 将来的なスクリーンリーダー対応を検討すること

### 6.5 言語・ローカライゼーション要件

- MVPは日本語のみ対応
- 将来的な英語対応を見越し、文字列の外部化を行うこと

### 6.6 セキュリティ要件

- 個人情報を一切収集しない (匿名プレイ)
- データはローカルに留まり、サーバー送信は行わない
- XSS等の基本的なWeb脆弱性対策を実施すること

## 7. 制約条件

### 7.1 技術的制約

- 対応ブラウザ: Chrome, Safari, Firefox, Edge の最新版およびその1つ前のメジャーバージョン
- 対応デバイス: PC (Windows, macOS, Linux), モバイル (iOS, Android)

### 7.2 法的・倫理的制約

- 実在する日本の政党名・政治家名を直接使用しない (ぼかし表現とする)
- 特定の政治的立場 (保守/リベラル, 親米/親中等) を過度に肯定・否定するような表現は避ける
- 複数視点を公平に提示する
- 史実や統計データを使用する場合は出典を明示する
- センシティブな歴史的事象 (戦争・植民地等) の取り扱いには十分配慮する

### 7.3 コンテンツ制約

- 暴力・性的表現は含めない (全年齢対象)
- 特定の民族・国籍・宗教を侮辱する表現は含めない
- ヘイトスピーチに該当する内容は含めない

## 8. 前提条件・依存事項

- ゲームの背景は現代日本の政治制度を基本とする
- プレイヤーは日本に一定の馴染みがある前提
- 衆議院定数465議席を前提とする
- 数値設計は2026年5月時点の日本の社会経済情勢を基本に置く

## 9. 将来の拡張要件

本ゲームの長期的なビジョンは、桃太郎電鉄のように **設定をカスタマイズして繰り返し遊べる政治シミュレーター** となることである。

| スタート位置 | 野党第一党 | 新興政党 / 野党 / 与党 / 連立与党 |

| シナリオ | 通常 (現代日本) | 戦後復興 / バブル後 / コロナ後 / 架空の危機 等 |

### 9.3 ゲームメカニクスの精緻化

| メカニクス | 内容 |
|---|---|
| 公約達成度システム | 施政方針演説で重点政策を宣言, 通過率を追跡し選挙時の信用度に反映 |
| 候補者の質システム | 党資金・党結束度から算出される候補者の質が議席に影響 |
| 選挙資金投入 | 党資金を選挙にどれだけ投じるかの戦略的選択 |

### 9.4 拡張性に関する実装方針

これら拡張は、MVPの内部設計で Configuration オブジェクト・データの宣言性・計算ロジックの抽象化を担保することにより、エンジン本体を書き換えずに実現する方針とする。データ構造には将来用フィールド (例: 政策・イベントの `educationalNote`) を予め含めておき、表示機能の追加だけで対応できるようにする。

## 11. 付録: 設計上の主要判断とその理由

| 判断 | 理由 |
|---|---|
| 野党第一党からスタート | 新興政党が10年で政権を取るのは非現実的。野党第一党からなら「成長→政権奪取」のアークが自然 |
| 期間10年固定 (MVP) | 中期スパンで複数選挙を経験できる。長すぎるとプレイ時間が膨大に |
| 党のイデオロギー固定 | 選挙ごとに変えられる「日和見」を排し、党の一貫性を戦略要素にする |
| 経済と財政を分けて管理 | ゲームの肝である「減税で支持率↑、財政↓」のトレードオフを明確にするため |
| 6党構成 (政党名はぼかし) | 実在政党をモデルにしつつ問題回避。多党制をシミュレートしながら複雑になりすぎない |
| 地方は6ブロック | 県単位は複雑すぎ、2分割は単純すぎる中間点 |
| トレンドとイベントの分離 | 「徐々に進行する変化」と「突発的な出来事」を明確に区別し、設計をクリーンに |
| ポジティブイベントの条件付け | 完全ランダムではなく政策の成果として現れる設計で、戦略性を出す |
| 拡張性を初期から確保 | 桃鉄スタイルの長期ビジョンのため、Config分離・データ宣言性・ロジック抽象化を初期から実装 |
| ターン前半/後半の性格分け | 現実の国会サイクルに対応させることでリアリティと教育的価値を向上 |
| 本予算と補正予算の区別 | 「平時の本予算 vs 非常時の補正予算」を選択することで予算判断にリアルな緊張感を与える |
| 定例イベントの導入 | 予算委員会・G7・国連総会などスケジュール固定の年中行事を入れ、現実の政治リズムを再現 |
| 教育サポート機能をMVP対象外 | MVPでは「動くゲームを最短で出す」ことを優先。注釈・用語集・振り返り画面は将来追加 |
| セーブ・ロード機能をMVP対象外 | 1プレイ1〜2時間で完結する想定のため。将来の長時間プレイ対応で追加 |
| 選挙ウェイトの簡略化 | MVPでは「地方支持率 × 経済情勢補正」の単純式に絞る。公約達成度や候補者の質は将来追加 |
| 連立交渉の自動化 | MVPでは距離が近い党から自動で連立を組む。プレイヤー選択や譲歩補正は将来追加 |
