## 1. 全体機能アーキテクチャ

### 1.1 機能一覧

| ID | 機能名 | 主な責務 |
|---|---|---|
| F-01 | ゲーム開始機能 | 新規ゲームの初期化 |
| F-02 | ターン進行機能 | フェーズ管理とターン遷移 |
| F-03 | 政策提出機能 | 政策の通過判定と効果反映 |
| F-04 | イベント対応機能 | イベント発生と対応処理 |
| F-05 | 予算配分機能 | 本予算・補正予算の配分 |
| F-06 | 選挙機能 | 選挙の実施と議席算出 |
| F-07 | 連立交渉機能 | 連立構成の決定 |
| F-08 | スコア算出・エンディング機能 | 最終評価と称号決定 |

### 1.2 機能間の依存関係

```
F-01 (開始)
  └─→ F-02 (ターン進行) [ループ x20]
        ├─→ F-04 (イベント発生) → 指標更新
        ├─→ F-03 (政策提出) → 指標更新
        ├─→ F-05 (予算配分) → 指標更新
        ├─→ トレンド進行 → 指標更新
        └─→ (選挙ターン) F-06 (選挙) → F-07 (連立)

最終ターン or 特殊終了
  └─→ F-08 (スコア算出・エンディング)
```

## 2. 共通定義

### 2.1 用語

| 用語 | 説明 |
|---|---|
| 国家の指標 | 支持率・経済・財政・外交安保・環境の5指標 |
| ターン番号 | 1〜20の整数 |
| 前半ターン | 奇数ターン (1, 3, ..., 19)。通常国会期間 (1-6月) |
| 後半ターン | 偶数ターン (2, 4, ..., 20)。臨時国会期間 (7-12月) |
| ゲーム年 | (ターン番号 + 1) ÷ 2 の切り捨て (1〜10) |

### 2.2 共通計算式

#### イデオロギー乖離 (政策と党)

```
イデオロギー乖離 = min(2, ( |経済軸の差| + |社会軸の差| + |外交軸の差| ) / 3)
```

各軸は -2〜+2 の整数値。

#### イデオロギー距離 (党と党)

連立交渉などで使用するユークリッド距離。

```
イデオロギー距離 = √( (経済差)² + (社会差)² + (外交差)² )
```

最大値は √48 ≈ 6.93。

#### 全国支持率 (地方加重平均)

```
全国支持率 = Σ (地方支持率 × 地方の議席シェア)
```

地方の議席シェアの合計は 1.0。

#### 指標値のクランプ

全指標は 0〜100 にクランプされる。

```
clamp(値) = max(0, min(100, 値))
```

## 3. ゲーム開始機能 (F-01)

### 3.1 機能概要

プレイヤーが党名と党のイデオロギーを設定し、新規ゲームを初期化する。

### 3.2 入力

| 項目 | 型 | 範囲 | 必須 |
|---|---|---|---|
| 党名 | 文字列 | 1〜20文字 | 必須 |
| 経済軸 | 整数 | -2〜+2 | 必須 |
| 社会軸 | 整数 | -2〜+2 | 必須 |
| 外交軸 | 整数 | -2〜+2 | 必須 |

### 3.3 処理内容

1. 入力バリデーション
2. ゲームConfig の生成 (MVPでは固定値)
3. 標準シナリオから初期 GameState を生成 (政党・地方ブロック・指標・トレンド進行度の初期化)
4. プレイヤー党にイデオロギーと党名を適用
5. ターン1 (前半) から開始

### 3.4 出力

- 初期化されたゲーム状態
- ターン1のイベントフェーズへの遷移

### 3.5 ビジネスルール

- 党名はNPC党 (民自党, 公明会, 改革維新党, 立憲党, 人民党) と重複不可
- イデオロギーは全軸0 (中道) を許容するが、UI上で「特徴のない党」警告を表示
- プレイヤー党の初期議席は 100〜120 議席 (野党第一党想定)
- 他党の初期議席はシナリオで定義 (民自党が与党想定で多めなど)

### 3.6 例外処理

| 状況 | 処理 |
|---|---|
| 党名空文字 | 入力エラー表示, 再入力要求 |
| 党名がNPC党と重複 | 警告表示, 別名を要求 |
| イデオロギー値が範囲外 | UI制限により発生しない |

## 4. ターン進行機能 (F-02)

### 4.1 機能概要

1ターン (6ヶ月) を以下のフェーズで進行させる。

### 4.2 ターンのフェーズ構成

```
[ターン開始]
   ↓
1. イベントフェーズ
   - 定例イベントの発生判定 (ターン種別に応じる)
   - ランダム/条件付きイベントの発生判定
   - トレンド進行
   - 連鎖イベントの処理
   - プレイヤー対応
   ↓
2. 政策フェーズ
   - 提出可能政策のリスト表示
   - プレイヤーが1〜3本選択
   - 各政策の通過判定 (F-03)
   - 通過政策の効果反映
   ↓
3. 予算配分フェーズ
   - 本予算 (前半) or 補正予算 (後半) の判定
   - プレイヤーが配分を決定 (F-05)
   ↓
4. 党運営フェーズ
   - 候補者擁立・派閥対応 (通常は自動)
   - 党結束度・連立健全度の自動更新
   ↓
5. 指標更新フェーズ
   - 全効果を統合して指標値を再計算
   - 0〜100 にクランプ
   ↓
6. (選挙ターンのみ) 選挙フェーズ
   - F-06 を実行
   - 結果次第で F-07 へ
   ↓
7. ターン履歴の記録
   ↓
8. 終了判定 → 次ターンへ or F-08 へ
[ターン終了]
```

### 4.3 前半ターン (奇数ターン) の特性

| 項目 | 内容 |
|---|---|
| 対応月 | 1月-6月 |
| 国会期間 | 通常国会 (150日) |
| 予算フェーズの対象 | 本予算 (翌年度) |
| 発生する定例イベント | 予算委員会, G7サミット, (党首討論) |
| 解散総選挙の発動可能性 | 6月 (任期内) |

### 4.4 後半ターン (偶数ターン) の特性

| 項目 | 内容 |
|---|---|
| 対応月 | 7月-12月 |
| 国会期間 | 国会閉会 + 臨時国会 |
| 予算フェーズの対象 | 補正予算 (任意) |
| 発生する定例イベント | 国連総会, 概算要求, (党首討論) |
| 解散総選挙の発動可能性 | 任期内なら任意 |

党首討論は前半・後半いずれかのターンで確率的に発生する (要件定義書 5.8 参照)。

### 4.5 ターン終了判定

以下のいずれかでゲーム終了:

- 20ターン目を終了 (通常終了)
- 特殊終了条件 (4.6) を満たす

### 4.6 特殊終了の発動条件

| 終了理由 | 条件 |
|---|---|
| 失脚エンド | 大型スキャンダル後、対応失敗で支持率が10未満 |
| 連立崩壊エンド | 与党時、連立健全度が10未満で連立離脱イベント発生 |
| 党分裂エンド | 党結束度が10未満の状態が3ターン継続 |

## 5. 政策提出機能 (F-03)

### 5.1 機能概要

プレイヤーが提出する政策の通過可否を判定し、通過した政策の効果を指標に反映する。

### 5.2 入力

プレイヤーが選択した政策のID (1〜3個)

### 5.3 処理内容

1. 選択された各政策について通過判定を実施
2. 通過した政策の効果を指標へ反映 (即時 or 遅延)
3. 通過しなかった政策は履歴に「否決」として記録
4. ターン履歴に記録

### 5.4 通過判定

#### 必要議席率

```
必要議席率 = 0.4 + 0.1 × 重要度 + 0.1 × イデオロギー乖離
```

特殊条件 (憲法改正等) を持つ政策は政策定義で上書きされる。

例:

- 重要度1, イデオロギー合致 (乖離0): 50%
- 重要度2, イデオロギー合致 (乖離0): 60%
- 重要度3, イデオロギー乖離2: 90%
- 憲法改正: 67% (2/3, 加えて国民投票発議が必要)

#### 実際の支持議席率

プレイヤー単独の議席に、連立相手の議席を加算する。連立相手は政策に対して一律で協力する。

```
実際の支持議席率 = (プレイヤー議席 + Σ 連立相手議席) / 衆議院定数
```

衆議院定数は GameConfig.houseTotalSeats から取得する (MVPでは 465)。

#### 党結束度によるボーナス

党結束度が 70 以上の場合、必要議席率を 0.05 引き下げる (詳細は 12.1 参照)。

#### 通過判定

実際の支持議席率 ≥ 必要議席率 のとき、政策は通過する。

### 5.5 効果反映ルール

- 即時効果 (timeliness = immediate): 当該ターンの指標更新フェーズで反映
- 遅延効果 (timeliness = short/medium/long): 規定ターン経過後に反映
- 副次効果 (イベント確率の変化など): 当該ターンの状態に反映
- 予算への影響: 党資金または国家予算から減算

#### 党結束度への影響

```
党結束度の変化 = -5 × イデオロギー乖離     (乖離 > 1 の場合)
党結束度の変化 = +2                       (乖離 ≤ 1 の場合)
```

### 5.6 ビジネスルール

- 1ターンに提出できる政策は1〜3本
- 同じ政策をゲーム中に複数回実行可能 (累積効果あり)
- 増税/減税の対になる政策は連続ターンで実行不可 (1ターン以上のクールダウン)
- 憲法9条改正は必要議席率2/3を満たした上で「国民投票」イベントが誘発され、結果次第で効果が適用される

### 5.7 例外処理

| 状況 | 処理 |
|---|---|
| 0本選択 | 政策フェーズをスキップ |
| 4本以上選択 | UI制限により発生しない |
| 党資金不足 | 該当政策の選択を不可とする |

## 6. イベント対応機能 (F-04)

### 6.1 機能概要

イベントを発生させ、プレイヤーの対応を反映する。

### 6.2 イベント発生判定

トリガー種別ごとに、各ターンのイベントフェーズで以下を判定する。

| トリガー種別 | 判定方法 |
|---|---|
| 時系列スクリプト (定例) | 該当ターンに必ず発生 |
| 政策連動 | 連動先政策の直近実行ありかつ確率判定で発生 |
| 条件付きランダム | 状態 (指標値等) が条件を満たすときに確率上昇判定 |
| 完全ランダム | 基礎確率による判定 |
| 連鎖 | 前のイベントから誘発される |

### 6.3 発生数の制限

1ターンに発生するイベント数の上限:

- 大規模イベント (scale = large): 最大1個
- 小規模イベント (scale = small): 最大3個
- 定例イベントは上記カウントとは別枠 (必ず発生)

複数候補が同時に発生した場合の優先順位: 定例 > 連鎖 > 政策連動 > 条件付き > 完全ランダム

### 6.4 選択肢処理

1. 即時効果 (選択前に確定する効果) がある場合は適用
2. 選択肢がある場合はプレイヤーに提示
3. プレイヤーの選択に応じて効果と副次効果を適用
4. 連鎖イベントが定義されている場合は次ターン以降のキューに追加
5. イベント履歴に記録

### 6.5 ビジネスルール

- 定例イベントは前半/後半ターンの種別と発生条件を満たせば必ず発生
- 同じイベントが10ターン以内に再発生することは原則禁止 (内部クールダウン)
- ポジティブイベントは関連政策が直近3ターン以内に実行されている場合のみ発生候補となる

### 6.6 例外処理

| 状況 | 処理 |
|---|---|
| イベント候補が0件 | フェーズをスキップ |
| 選択肢提示時の不整合 | デフォルト選択肢を自動採用 |

## 7. 予算配分機能 (F-05)

### 7.1 機能概要

ターン種別に応じた予算 (本予算 / 補正予算) の配分を行う。

### 7.2 前半ターン: 本予算配分

1. 歳入を算出 (前ターンの経済指標と税制政策の結果から)
2. プレイヤーが主要6分野に配分を割り振る
3. 制約: 合計が歳入 + 許容赤字 以下であること
4. 配分結果を該当指標に反映

#### 歳入の計算

```
歳入 = ベース歳入 × (1 + 経済指標補正) + 税制政策補正
```

具体値はゲームバランス設計書で確定する。

### 7.3 後半ターン: 補正予算

1. 必要性を表示 (災害・経済ショックの発生有無で推奨度が変動)
2. プレイヤーが組むか組まないかを選択
3. 組む場合、規模を指定
4. 効果を反映 (財政指標への悪影響と引き換えに緊急対応力)

### 7.4 配分先と指標への影響

| 配分先 | 主な指標への影響 |
|---|---|
| 税制財政 | 財政+ (徴税効率) |
| 経済産業 | 経済+, 雇用+ |
| 社会保障 | 支持率+ (特に高齢層), 財政- |
| 教育科技 | 経済+ (長期), 環境+ (若干) |
| 環境エネルギー | 環境+, 経済- (規制増の場合) |
| 外交防衛 | 外交安保+ |

各係数はゲームバランス設計書で確定する。

### 7.5 ビジネスルール

- 本予算は前半ターンに1回提案
- 補正予算は後半ターンに任意 (組まない選択も可)
- 配分未指定の分野は最低限の現状維持予算が自動で適用
- 大規模補正予算は財政への悪影響と引き換えに緊急性ある対応力が増す

### 7.6 例外処理

| 状況 | 処理 |
|---|---|
| 歳入不足の本予算 | エラー, 再入力要求 |
| 補正予算入力なし (後半) | スキップ扱い |

## 8. 選挙機能 (F-06)

### 8.1 機能概要

衆議院議員総選挙を実施し、各党の議席数を算出する。

### 8.2 選挙発動条件

#### 野党時

各ターン終了時に確率で発動。前回選挙からの経過ターン数で確率が増加する。

| 経過 | 発動確率 |
|---|---|
| 1年 (2ターン) | 25% |
| 2年 (4ターン) | 50% |
| 3年 (6ターン) | 75% |
| 4年 (8ターン) | 100% (任期満了) |

#### 与党時

プレイヤーが任意のタイミングで「解散総選挙」を選択可能。前回選挙から最大8ターンで自動発動。

### 8.3 議席計算

#### 党のウェイト

地方支持率を主軸に、経済情勢補正で現職効果を反映する。

```
党のウェイト = 地方支持率 × (1 + 経済情勢補正)
```

#### 経済情勢補正 (現職効果)

```
与党の経済補正   = +(経済指標 - 50) × 与党係数
連立党の経済補正 = +(経済指標 - 50) × 連立党係数
野党の経済補正   = -(経済指標 - 50) × 野党係数
```

各係数の関係: 与党係数 > 連立党係数 > 野党係数 ≥ 0。具体値はゲームバランス設計書で確定する。

経済が良ければ与党有利、悪ければ与党不利。

#### 地方ブロックごとの議席算出

```
党の地方議席 = round( 地方の総議席 × 党のウェイト / 全党のウェイト合計 )
```

端数は最大ウェイトの党に割り当てて調整する。

#### 全国議席の集計

各地方ブロックの議席を合算する。

### 8.4 与党/野党の判定

- 単独過半数 (233議席以上) を獲得した党が与党 (単独政権)
- 過半数未達の場合、第一党を起点に連立交渉 (F-07) を実行

### 8.5 与党時の解散戦略 (情報提示)

プレイヤーの判断を助けるため、解散時に以下を表示する。

| 情報 | 内容 |
|---|---|
| 現在の支持率 | 解散直後の選挙で有利になる目安 |
| 任期残期間 | 強制選挙までの残ターン数 |
| 経済情勢 | 現職効果の強さ |
| 不信任案リスク | 党結束度・スキャンダル状況から推測 |

最終判断はプレイヤーが行う。

### 8.6 ビジネスルール

- 選挙後の最初のターンは「特別国会」相当として、定例イベント以外のイベントが半減
- 選挙結果に応じて与党/野党と連立の構成を更新

### 8.7 例外処理

| 状況 | 処理 |
|---|---|
| 議席合計が定数に合わない | 端数調整 (最大ウェイトの党) |
| 全党のウェイトが0 | 防御的に均等分配 |
| プレイヤー党の議席が0 | 失脚エンドへ |

## 9. 連立交渉機能 (F-07)

### 9.1 機能概要

選挙結果が単独過半数に達しない場合、他党との連立を自動的に構成する。

### 9.2 連立必要判定

第一党の議席が 232 以下のときに連立交渉を発動する。

### 9.3 交渉プロセス

1. 連立候補党をイデオロギー距離順にソート
2. 距離が近い順に連立成立判定を実施
3. 過半数 (233議席) に達するまで連立メンバーを追加

### 9.4 連立成立判定

```
連立成立率 = max(0, 1 - イデオロギー距離 / 7)
```

成立しなかった候補はスキップし、次の候補に移る。

### 9.5 連立健全度の初期値

```
連立健全度の初期値 = 80 - イデオロギー距離 × 5
```

複数党との連立の場合は、各連立相手とのイデオロギー距離の平均値を使用する。

### 9.6 連立健全度の変動

| 要因 | 変動量 |
|---|---|
| 連立相手の意向と離れた政策の実行 | -5 (毎政策) |
| 連立相手の意向に沿う政策の実行 | +1 (毎政策) |

### 9.7 連立健全度に応じたイベント発生

| 連立健全度 | 影響 |
|---|---|
| 70以上 | 連立円滑 |
| 30〜69 | 通常状態 |
| 10〜29 | 連立緊張イベントが発生 |
| 10未満 | 連立崩壊イベント, 失脚エンドへの導線 |

### 9.8 ビジネスルール

- 連立交渉は選挙後の1ターン以内に確定
- 連立解消は連立健全度<10の連立崩壊イベントで発生
- 野党時, 与党側からプレイヤー党が連立に誘われる場合もある

### 9.9 例外処理

| 状況 | 処理 |
|---|---|
| どの党とも連立成立せず | 第一党による少数与党 (政策通過が困難) |

## 10. スコア算出・エンディング機能 (F-08)

### 10.1 機能概要

ゲーム終了時に総合スコアを算出し、エンディング種別と称号を決定する。

### 10.2 ゲーム終了判定

- 通常終了: 20ターン経過
- 特殊終了: 4.6節の条件

### 10.3 スコア計算

#### 5指標の平均

```
指標平均 = (支持率 + 経済 + 財政 + 外交安保 + 環境) / 5
```

#### 党理念達成度

自党イデオロギーに沿った政策の実行割合。

```
党理念達成度 = (イデオロギー乖離 < 1 の通過政策数 / 通過政策の総数) × 100
```

通過政策がゼロの場合は0とする。

#### 在任比率

```
在任比率 = 在任ターン数 / 総ターン数 × 100
```

#### 総合スコア

```
総合スコア = 0.5 × 指標平均 + 0.3 × 党理念達成度 + 0.2 × 在任比率
```

### 10.4 エンディング種別の判定

特殊終了が優先される。通常終了の場合は条件に応じて以下を判定する。

| 称号 | 条件 |
|---|---|
| 歴史的名宰相 | 全指標が80以上 |
| 経済再生の立役者 | 経済が75以上, かつ財政が75以上 |
| グリーンリーダー | 環境が85以上 |
| 信念の人 | 党理念達成度が90以上 |
| 老獪な政治家 | 在任比率が80以上, かつ指標平均が60未満 |
| 標準エンディング | 上記いずれにも該当しない |

特殊終了: 失脚エンド / 連立崩壊エンド / 党分裂エンド

### 10.5 結果表示

ゲーム終了後、以下を提示する:

- 最終指標の値
- 受賞称号
- 在任ターン数

### 10.6 例外処理

| 状況 | 処理 |
|---|---|
| 政策履歴が空 | 党理念達成度を0として扱う |
| 全指標0 | 失脚エンドとして処理 |

## 11. トレンド進行ロジック

### 11.1 概要

5本のトレンドが毎ターン自動進行し、指標に持続的な影響を与える。

### 11.2 トレンドの進行式

```
新しい進行度 = clamp(現在の進行度 + 基準進行量 × 抑制係数, 0, 100)
```

### 11.3 抑制係数

各トレンドには対抗政策が定義されており、それらが直近5ターン以内に実行されている場合、進行が抑制される。

```
抑制係数 = 抑制値    (該当する対抗政策が直近5ターン以内に1つ以上実行されている場合)
抑制係数 = 1.0      (それ以外)
```

抑制値 (0〜1の範囲) はゲームバランス設計書で確定する。

### 11.4 進行度に応じた指標への影響

```
当ターンの指標への影響 = 進行度 / 100 × 基準効果
```

進行度が高いほど指標への影響が強くなる。

### 11.5 ビジネスルール

- 全トレンドは進行度0でゲーム開始、100で最大影響
- 進行したトレンドは原則戻らない (対抗政策で進行を抑えることは可能)
- 米中対立トレンドは外部シナリオの影響を受ける

## 12. 状態 (党結束度・連立健全度) の管理

### 12.1 党結束度

党内派閥の結束度を 0〜100 で管理する。

#### 変動要因

| 要因 | 変動量 |
|---|---|
| 党理念に沿った政策の実行 | +2 |
| 党理念から外れた政策の実行 | -5 × イデオロギー乖離 |

#### 結束度に応じた影響

| 結束度 | 影響 |
|---|---|
| 70以上 | 党内一致, 政策通過にボーナス (必要議席率 -0.05) |
| 40〜69 | 通常状態 |
| 20〜39 | 派閥対立イベント発生確率増 |
| 10〜19 | 議席離脱リスク (派閥対立イベントの確率さらに増) |
| 10未満 | 党分裂リスク。10未満の状態が3ターン継続すると党分裂エンドに到達 (4.6 参照) |

結束度70以上のボーナスは、政策通過判定 (5.4) における必要議席率を 0.05 引き下げる形で適用する。

### 12.2 連立健全度

連立相手との関係を 0〜100 で管理する (与党時のみ)。

詳細は9章 (連立交渉機能) を参照。

## 13. 付録: 主要計算式集約

### 政策通過判定

```
必要議席率 = 0.4 + 0.1 × 重要度 + 0.1 × イデオロギー乖離
             (党結束度 ≥ 70 のとき -0.05)

実際の支持議席率 = (プレイヤー議席 + Σ 連立相手議席) / 衆議院定数

衆議院定数 = GameConfig.houseTotalSeats (MVP: 465)
```

### イデオロギー関連

```
イデオロギー乖離 = min(2, (|経済差| + |社会差| + |外交差|) / 3)

イデオロギー距離 = √((経済差)² + (社会差)² + (外交差)²)
```

### 選挙関連

```
党のウェイト = 地方支持率 × (1 + 経済情勢補正)

党の地方議席 = round(地方の総議席 × 党のウェイト / 全党のウェイト合計)
```

### 連立関連

```
連立成立率 = max(0, 1 - イデオロギー距離 / 7)

連立健全度の初期値 = 80 - イデオロギー距離 × 5
```

### 最終スコア

```
総合スコア = 0.5 × 指標平均 + 0.3 × 党理念達成度 + 0.2 × 在任比率

党理念達成度 = (イデオロギー乖離 < 1 の通過政策数 / 通過政策の総数) × 100
```

### その他

```
全国支持率 = Σ (地方支持率 × 地方の議席シェア)
```
